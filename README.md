# Проект Daria Kulchikhina

В репозитории реализован приватный кабинет для трёх ролей:

- дизайнер/администратор (управление проектами, подрядчиками, контентом),
- клиент (просмотр страниц проекта),
- подрядчик (просмотр закреплённых задач).

Ключевое обновление структуры: создание проекта выполняется через мастер с выбором сценария, после чего сервер автоматически создаёт полный каркас проекта (основные страницы + стартовый roadmap по шаблону).

## Обязательное правило по авторизации

- Никогда не внедрять и не предлагать PIN-авторизацию как способ входа.
- Любые изменения в auth-flow должны использовать только согласованные способы авторизации, без PIN-входа.
- Если возникает неоднозначность по способу входа — сначала согласовать с владельцем проекта, без самостоятельного добавления PIN-сценариев.

## Быстрый запуск

```bash
pnpm install
pnpm dev
```

Приложение запускается на `http://localhost:3000`.

## Безопасный деплой (с fallback)

Для стабильного прод-деплоя добавлен скрипт с автоматическим резервным сценарием:

1. синхронизирует исходники на сервер,
2. пытается собрать проект на сервере,
3. если серверная сборка падает — собирает локально и отправляет `.output` в прод,
4. перезапускает `pm2` и проверяет health-check.

Запуск:

```bash
pnpm deploy:safe
```

Готовая команда для прода (с уже зашитыми параметрами сервера):

```bash
pnpm deploy:safe:prod
```

Ускоренный запуск без preflight-проверок:

```bash
pnpm deploy:safe:prod:fast
```

Dry-run для проверки деплоя без перезапуска процесса:

```bash
pnpm deploy:safe:prod:dry-run
```

`dry-run` выполняет sync + install + build (и fallback при необходимости), но не делает `pm2 restart` и не запускает health-check.

Во всех режимах скрипт печатает длительность этапов (`[time] ...s`), чтобы быстро видеть узкие места деплоя.
Также метрики записываются в `logs/deploy-metrics.log` (timestamp, режим, статус, длительности этапов).

Просмотр последних записей:

```bash
pnpm deploy:metrics
```

Онлайн-просмотр:

```bash
pnpm deploy:metrics:follow
```

Preflight-only (только проверки окружения, без sync/build/restart):

```bash
pnpm deploy:safe:prod:preflight
```

Проверяет локальные утилиты, SSH-доступ, наличие `node/pnpm/pm2` на сервере и права на `DEPLOY_PATH`.

Если нужно пропустить preflight в любом сценарии, можно задать `NO_PREFLIGHT=1`.

Переопределяемые переменные окружения:

- `DEPLOY_HOST` (по умолчанию `daria-deploy`)
- `DEPLOY_PATH` (по умолчанию `/opt/daria-nuxt`)
- `APP_NAME` (по умолчанию `daria-nuxt`)
- `HEALTHCHECK_URL` (по умолчанию `http://152.53.176.165:3000/admin`)
