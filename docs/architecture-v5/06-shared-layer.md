# 6. Слой Shared (Общие контракты)

Директория `shared/` содержит код, который импортируется одновременно в `app/` (фронтенд), `server/` (бэкенд) и `messenger/` (микросервис). Это гарант консистентности данных между всеми слоями.

## Структура директории `shared/`

- **`types/`**: 
  - Общие TypeScript типы и интерфейсы.
  - Zod-схемы для валидации запросов и ответов API.
  - Экспортируемые типы из схем Drizzle (например, `type User = typeof users.$inferSelect`).
- **`constants/`**: 
  - Общие справочники и перечисления (enum).
  - Конфигурации страниц (`pages.ts`).
  - Мета-поля и роли (`profile-fields.ts`).
- **`utils/`**: 
  - Универсальные хелперы и чистые функции без привязки к среде исполнения.
  - Форматирование данных, дат, парсинг специфичных строк.

## Строгие правила слоя Shared
1. **Изоляция окружения**: Файлы внутри `shared/` **не должны** импортировать ничего из `app/` (браузер) или `server/` (Node.js).
2. **Платформонезависимость**: Запрещено использование специфичных API (таких как `window`, `localStorage`, `fs`, `path`).
3. **Единый источник истины**: Любые константы статусов или ролей должны объявляться здесь, а не хардкодиться в компонентах или контроллерах.
