# 5. Ключевые Архитектурные Паттерны

## 5.1. Ticket-Based WebSocket Auth
- Фронт запрашивает: `POST /api/chat/ws-ticket`.
- Nuxt проверяет HMAC-cookie, генерирует токен и сохраняет в Redis: `SET ws_ticket:<token> {userId: 1} EX 30`.
- Фронт открывает WS: `wss://ws.daria.com?ticket=<token>`.
- WS-сервер (messenger) читает Redis. Если токен есть — соединение открывается. БД отдыхает.

## 5.2. Управление ключами E2EE (Защита от кражи)
1. При первой настройке E2EE пользователь вводит пароль (Master Key).
2. С помощью PBKDF2 (и соли `master_key_salt` из БД) пароль превращается в KEK.
3. Браузер (WebCrypto API) генерирует ключи. `encrypted_private_key` уходит в БД как бекап.
4. При новом логине юзер вводит пароль, фронт скачивает зашифрованный ключ из БД и расшифровывает локально.

## 5.3. E2EE Шифрование файлов и MinIO
1. Юзер прикрепляет фото. Фронтенд локально шифрует файл случайным AES ключом.
2. Фронт запрашивает S3 Presigned URL и загружает зашифрованный Blob в MinIO.
3. Ссылка на MinIO записывается в таблицу `attachments`. AES ключ шифруется открытым ключом получателя и отправляется в текстовом сообщении.

## 5.4. Офлайн режим и Оптимистичные блокировки (OCC)
Внедрено для предотвращения перезаписи данных при плохой связи. БД проверяет `version` записи перед любым UPDATE. Если версия на сервере старше клиентской — возвращается `409 Conflict`. Удаленные записи помечаются `deleted_at = NOW()`, чтобы оффлайн-клиент при синхронизации понял, что объект нужно убрать из UI.

## 5.5. Синхронизация состояний UI (Reactive Sync)
Бэкенд обновляет статус проекта и кидает в Redis: `PUBLISH entity_updates`. Мессенджер прокидывает событие через WebSocket на клиент, где Pinia-стор реактивно обновляет UI дашборда без перезагрузки.

## 5.6. Cursor-Based Пагинация для Мессенджера
Для истории чата используется строго курсорная пагинация. Запрос: `GET /api/messages?before=<created_at>`. Это гарантирует O(1) время выборки истории по B-Tree индексу.

## 5.7. Headless UI / ChatOps
Фронтенд читает `type` системного сообщения в таблице `messages` (напр. `widget_moodboard_approval`), берет данные из колонки `payload` (JSONB) и рендерит Vue-компонент прямо внутри ленты чата.

## 5.8. Headless Design System
- **Реактивный CSSOM**: При движении ползунка `useThemeStore` мгновенно обновляет CSS-переменные (`setProperty`).
- **Persistence**: Сохранение токенов идет в БД (`page_configs.ts`).
- **Кэширование**: Бэкенд кэширует тему в Redis для мгновенного SSR рендера новым пользователям.
