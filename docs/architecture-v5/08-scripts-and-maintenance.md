# 8. Скрипты и Обслуживание (scripts/)

Каталог `scripts/` содержит утилиты для оркестрации, безопасного деплоя, миграции данных и сидирования базы. Эти скрипты — основа нашего инфраструктурного пайплайна на сервере.

## 8.1. Деплой и Инфраструктура

- **`scripts/deploy-safe.sh`**: Главный скрипт развертывания.
  1. Выполняет проверку чистоты Git (`git status --porcelain`).
  2. Делает пуш в `origin main`.
  3. На сервере выполняет `pnpm build` для Nuxt и Messenger.
  4. Запускает `pm2 reload`.
- **`scripts/fix.sh`**:
  - Адаптер для лечения конфликтов ESM/CJS, которые иногда возникают во Vue SSR в рантайме Nitro. Прогоняется автоматически после сборки.
- **`scripts/export-messenger-standalone.sh`**: Скрипт сборки и изоляции артефактов мессенджера.
- **`scripts/restore-last-snapshot.sh`**: Откат к предыдущему рабочему состоянию БД.

## 8.2. Дата-миграции (`scripts/migrate-*.mjs`)

Скрипты `migrate-` предназначены для *миграции данных* (Data Migrations), когда меняется структура, требующая бизнес-логики, которую нельзя описать простым SQL файлом в `drizzle-kit`:

- `scripts/migrate-json-to-pg.mjs`: Исторический скрипт переезда старых JSON-баз на реляционную PostgreSQL.
- `scripts/migrate-gallery-images.mjs`, `scripts/migrate-gallery-properties.mjs`: Трансформации сущностей галереи (тэги, картинки, свойства).
- `scripts/migrate-legal-chunks.mjs`, `scripts/migrate-add-documents.mjs` и др.: Заполнение новых колонок на основе существующих данных.

## 8.3. Сидирование (Seeding)

Используется для разработческой среды и демонстрации:

- **`scripts/seed-messenger-agents.ts`**: Наполнение базы мессенджера агентами и начальной конфигурацией.
- На проде сидеры не запускаются!
