# 8. Скрипты и Обслуживание (scripts/)

Каталог `scripts/` содержит утилиты для оркестрации, безопасного деплоя, миграции данных и сидирования базы. Эти скрипты — основа нашего инфраструктурного пайплайна на сервере.

## 8.1. Деплой и Инфраструктура
- **`deploy-safe.sh`**: Главный скрипт развертывания.
  1. Выполняет проверку чистоты Git (`git status --porcelain`).
  2. Делает пуш в `origin main`.
  3. На сервере выполняет `pnpm build` для Nuxt и Messenger.
  4. Запускает `pm2 reload`.
- **`repair-nitro-vue-runtime.mjs` / `fix.sh`**: 
  - Адаптер для лечения конфликтов ESM/CJS, которые иногда возникают воVue SSR в рантайме Nitro. Прогоняется автоматически после сборки.
- **`export-messenger-standalone.sh`**: Скрипт сборки и изоляции артефактов мессенджера.
- **`restore-last-snapshot.sh`**: Откат к предыдущему рабочему состоянию БД.

## 8.2. Дата-миграции (`migrate-*.mjs`)
Скрипты `migrate-` предназначены для *миграции данных* (Data Migrations), когда меняется структура, требующая бизнес-логики, которую нельзя описать простым SQL файлом в `drizzle-kit`:
- `migrate-json-to-pg.mjs`: Исторический скрипт переезда старых JSON-баз на реляционную PostgreSQL.
- `migrate-gallery-*`: Трансформации сущностей галереи (тэги, картинки, свойства).
- `migrate-legal-chunks.mjs`, `migrate-add-documents.mjs` и др.: Заполнение новых колонок на основе существующих данных.

## 8.3. Сидирование (Seeding)
Используется для разработческой среды и демонстрации:
- **`seed-demo-v2.mjs` / `seed-demo-ivanov.mjs`**: Наполнение базы полностью готовым фейковым дизайн-проектом (Клиент Иванов, задачи, фазы ремонта, чаты).
- На проде сидеры не запускаются!
