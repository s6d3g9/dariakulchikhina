🔔 Система уведомлений реализовано
API: GET /api/notifications · POST /api/notifications/[id]/read · POST /api/notifications/read-all
Уведомления хранятся в таблице notifications в PostgreSQL.
Параметр / полеОписание
roleadmin | customer | contractor | all — кому показывается
projectIdПривязка к проекту (для клиентских уведомлений)
contractorIdПривязка к подрядчику (для уведомлений подрядчику)
isReadboolean — прочитано
GET ?page=&limit=&unread=1Пагинация, фильтр только непрочитанных
Каждая роль видит только свои уведомления (middleware-фильтр в API) «Отметить все прочитанными»
📄 PDF-экспорт реализовано
API: POST /api/pdf/[projectId] body: { type: "tz" | "roadmap" | "payment" }
Генерация через PDFKit (Node.js). Файл сохраняется в public/uploads/pdf/ и отдаётся URL.
typeСодержимое PDF
tzТехническое задание проекта — разделы, вопросы, ответы из page_content
roadmapДорожная карта — все этапы с датами, статусами, заметками
paymentСписок платежей с историей статусов и итогами
Лог экспорта сохраняется в project.profile.exportLog Кнопка «pdf» в AdminPayments сразу вызывает POST

Все данные — только для чтения, редактирование недоступно.

Уведомления — только для своего проекта (role = customer | all, projectId = свой).
