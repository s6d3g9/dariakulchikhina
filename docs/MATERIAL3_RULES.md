# Правила построения Material Design 3 (M3)

Основано на официальной спецификации: https://m3.material.io/

## 1. Цветовая система и Тональные палитры (Tonal Palettes)
В Material 3 цвет строится на основе динамических тональных палитр, а не жестко зафиксированных HEX-значений. Каждому цвету (Primary, Secondary, Tertiary, Error, Neutral) выделяется шкала от 0 (черный) до 100 (белый).

### Разрешительные правила (DOs):
- **ИСПОЛЬЗОВАТЬ семантические токены:** Назначать цвета по их роли: `primary`, `on-primary`, `primary-container`, `on-primary-container`. Использовать `surface-container` (lowest, low, normal, high, highest) для разделения слоев.
- **ИСПОЛЬЗОВАТЬ тональную элевацию:** Выделять карточки и контейнеры не только тенями (drop-shadow), но и легким высветлением (в светлой теме) поверхностей для имитации высоты (Tonal Elevation).
- **ИСПОЛЬЗОВАТЬ контрастные пары:** Текст или иконки должны всегда использовать соответствующий `on-*` цвет (например, текст на `primary` фоне должен использовать `on-primary`).

### Запретительные правила (DON'Ts):
- **НЕЛЬЗЯ использовать прямые HEX цвета в интерфейсе:** Все цвета должны проходить через переменные темы.
- **НЕЛЬЗЯ ограничиваться только белым фреймом / черным текстом:** Применять `surface` и `surface-dim`.
- **НЕЛЬЗЯ использовать обводку (border) для разделения плоскостей по умолчанию:** Бордеры используются только для специфичных Outline компонентов (Outlined Card, Outlined Button).

## 2. Типографика (Typography Scale)
Система разделена на 5 ролей: Display, Headline, Title, Body, Label (каждая имеет размеры Small, Medium, Large).

### Разрешительные правила (DOs):
- **ИСПОЛЬЗОВАТЬ Display и Headline для акцентов:** Названия разделов или ключевые метрики.
- **ИСПОЛЬЗОВАТЬ Title для карточек и списков:** Иерархия должна считываться мгновенно.
- **ИСПОЛЬЗОВАТЬ Label для кнопок, бейджей и мелких подписей.**
- **СОБЛЮДАТЬ высоту строки (Line Height):** Material 3 использует просторные line-height для удобочитаемости (например, `line-height: 24px` для 16px Body).

### Запретительные правила (DON'Ts):
- **НЕЛЬЗЯ использовать произвольные размеры шрифтов (например, 15px или 17px).**
- **НЕЛЬЗЯ использовать слишком плотный межстрочный интервал (line-height < 1.25) для длинных текстов.**

## 3. Формы (Shape Scale)
В M3 используется строгая шкала скруглений (Corner Radius):
- **None (0px):** полноэкранные элементы.
- **Extra Small (4px):** Снэкбары, Tooltips, текстовые поля (Filled).
- **Small (8px):** Мелкие чипсы, меню.
- **Medium (12px):** Карточки, диалоги (исторически 12px, M3 иногда переходит к 16px/28px).
- **Large (16px):** Floating Action Buttons (FAB), карточки.
- **Extra Large (28px):** Крупные диалоговые окна (AlertDialog), Bottom Sheets, Navigation Drawer.
- **Full / Pill (100px / 50%):** Кнопки (Button), Chips, Search Bar.

### Разрешительные правила (DOs):
- **ИСПОЛЬЗОВАТЬ строгую шкалу:** Если элемент круглый — это Pill. Если это стандартная карточка — 12px или 16px. Диалоги стали более округлыми — 28px.
- **ИСПОЛЬЗОВАТЬ асимметричные скругления:** Допустимо для выделения активных состояний (например, скругление справа больше, чем слева).

### Запретительные правила (DON'Ts):
- **НЕЛЬЗЯ использовать произвольные радиусы (например, 3px, 5px, 14px, 20px).**
- **НЕЛЬЗЯ делать стандартные кнопки квадратными (0px), если того не требует специфичный брендинг, конфликтующий с M3.** M3 Button = Full/Pill shape.

## 4. Размеры и отступы (Layout & Spacing)

### Разрешительные правила (DOs):
- **ИСПОЛЬЗОВАТЬ сетку с шагом 4px и 8px:** Отступы 4, 8, 12, 16, 24, 32, 48 и т.д.
- **ОБЕСПЕЧИВАТЬ минимальный размер области нажатия (Touch Target):** 48x48 dp. Сам визуальный элемент может быть меньше (например, иконка 24x24 px внутри области 48x48 px).

### Запретительные правила (DON'Ts):
- **НЕЛЬЗЯ размещать интерактивные элементы вплотную друг к другу (< 8px).**
- **НЕЛЬЗЯ использовать отступы, не кратные 4.**

## 5. Элевация (Elevation) и Тени
M3 ушел от глубоких тяжелых теней M2.
Уровни: 
- Level 0: Нет тени, базовый фон.
- Level 1: Tonal Light + очень слабая тень.
- Level 2: Tonal Tweak + слабая тень (Navigation bars).
- Level 3: Dropdown menus.
- Level 4/5: Dialogs & Bottom Sheets.

### Разрешительные правила (DOs):
- **ИСПОЛЬЗОВАТЬ Tonal color поверх тени,** чтобы показать высоту.
- **Допускать отсутствие рамок** вокруг карточек за счет цветового отделения `surface-container` от `surface`.

### Запретительные правила (DON'Ts):
- **НЕЛЬЗЯ использовать сильные (`0px 8px 24px rgba(0,0,0,0.5)`) жесткие тени.** Дизайн должен быть плоским и легким (flat & tonal).

## 6. Отказ от бесполезных инфо-панелей (Zero-Fluff UI)
Интерфейс должен быть функциональным, а не нарративным. Не нужно объяснять пользователю очевидные вещи.

### Разрешительные правила (DOs):
- **ИСПОЛЬЗОВАТЬ только функциональные кнопки-решения** (например, фильтры, закрытие, добавление).
- **ИСПОЛЬЗОВАТЬ компактные лейблы или иконки** вместо длинных объясняющих предложений.

### Запретительные правила (DON'Ts):
- **НЕЛЬЗЯ показывать пояснительные инфо-таблички, "шапки" с описанием разделов**, если они не несут прямого функционала (например, текст "Фото, стикеры, файлы собраны по разделам" — избыточен). Оставлять только интерактивные элементы управления.
